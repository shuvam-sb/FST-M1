Application: Native App   [Activity 1 -> 3 ]
We start with the To-Do List app. Please download the apk file from here and drag and drop it into the emulator/device to install it: To-Do List App
List of Activities:

Activity 1 
Goal: Use the To-Do list app to create a list of tasks that need to be completed.
Open the app.
Click the button to add a new task.
Add the following tasks:
Complete Activity 1 with priority high
Complete Activity 2 with priority medium
Complete Activity 3 with priority low
After each task is added, the Save button should be clicked.
Write an assertion to ensure all three tasks have been added to the list.

Activity 2
Goal: Edit a task to add a deadline.
Open app.
Long press the first task in the list to edit its details.
Set a deadline for the task to be completed by next Saturday.
Save the edited details and add assertion to confirm that the deadline is set.

Activity 3
Goal: Mark all tasks as complete.
Open the app.
Mark the first two tasks as complete.
Long press the third task to edit it. In the edit details, slide the progress bar to 50% and save the details.
Click the option icon and select “Completed tasks”.
Make an assertion to ensure that only 2 tasks as shown here.




Application: Google Chrome  [Activities 4 -> 6]

Next, we use the Chrome browser app on a mobile device to test with web pages.
List of Activities:

Activity 4 
Goal: Opening a page on the browser and testing a to-do list page 
Open the Chrome app on your device.
Go to the following URL: https://training-support.net/webelements
Scroll to find the To-Do List card and click it.
Once the page loads, find the input field on the page and enter the following three tasks:
Add tasks to list
Get number of tasks
Clear the list
Click on each of the tasks added to strike them out.
Add assertions to verify the count of the number of tasks in the list. Hint: There are 2 tasks already present in the list.
